## Security Threat Model

| Threat | Severity | Mitigation |
|--------|----------|------------|
| T-01-01: Native dependency install or runtime step executes unexpected build behavior on developer machines. | medium | Pin the accepted package version, record install/runtime behavior, and do not add curl/download scripts to normal install. |
| T-01-02: Failed TUI startup leaves terminal raw/alternate-screen state broken. | medium | The prototype must call renderer cleanup in `finally`/signal paths and document the cleanup result. |
| T-01-03: Runtime fallback silently swaps away from the user-selected OpenTUI decision. | low | Do not switch libraries automatically. If OpenTUI friction is unacceptable, stop with a summary recommending reconsideration of `blessed-contrib`. |
</threat_model>
