## Security Threat Model

| Threat | Severity | Mitigation |
|--------|----------|------------|
| T-02-01: Dashboard rewrite bypasses existing peer manager safety checks for kill/log operations. | high | Use existing `killPeer()`, `listPeers()`, and `readPeerLog()` only; do not implement process or filesystem control in UI modules. |
| T-02-02: Rendering unbounded log content harms terminal responsiveness. | medium | Keep log tails bounded and route through `readPeerLog(peer.id, N)` with fixed N values. |

</threat_model>
